Web-приложение (MVP) реализует простую систему учета успеваемости студентов по дисциплине обучения.
Приложение разработано с использованием технологий Entity Framework и ASP.NET Web Forms (выбор технологии обусловлен сжатыми сроками подготовки
зачетного задания, Web Forms позволяют воспользоваться преимуществами rapid application development).

Веб-сайт состоит 4 страниц на основе мастер-страницы. Навигация на сайте реализована на основе мастер-страницы.

Страницы и функционал:
1. Стартовая страница - страница просмотра оценок студентов - "View grades"
Пользователь выбирает с помощью календаря дату начала и конца периода
Реализована проверка выбора дат
По нажатию кнопки выгружаются оценки студентов, полученные в течение заданного периода, а также их сумма и среднее.
Возможные направления дальнейшей работы:
- настройка сортировки выгруженных данных
- настройка выбора/поиска студентов, для которых выгружаются оценки, по фамилии/имени (и/или начальным буквам фамилии/имени)

2. Страница добавления данных о новом студенте - "Add new student"
Пользователь вводит фамилию, имя и дату записи на курс (с помощью календаря) для нового студента (ID студента генерируется СУБД автоматически)
Реализована проверка заполнения обязательных полей
По нажатию кнопки данные загружаются в БД, на вэб-форме отражается сообщение об успешной загрузке данных
Если данные загрузить не удалось, возникает сообщение об ошибке
Возможные направления дальнейшей работы:
- настройка сортировки списка студентов
- настройка выбора/поиска студента, которому выставляется оценка, по фамилии/имени (и/или начальным буквам фамилии/имени)
- настройка редактирования информации о "старых" студентах

3. Страница добавления новых оценок - "Add grade"
Пользователь выбирает с помощью календаря дату для выставления оценки и выбирает студента, которому выставляется оценка, в GridView,
а также вводит оценку (от 1 до 100)
Реализована проверка заполнения обязательных полей
Реализована проверка, что оценка задана корректно (число от 1 до 100)
По нажатию кнопки данные загружаются в БД, на вэб-форме отражается сообщение об успешной загрузке данных
Возможные направления дальнейшей работы:
- реализовать на одной странице просмотр старых оценок и ввод новой оценки

4. Страница просмотра 5 лучших и 5 худших студентов - "Top and bottom results"
Пользователь выбирает с помощью календаря дату начала и конца периода, за который вбираются лучшие и худшие студенты
Пользователь выбирает, как будут определены лучшие и худшие студенты: по сумме баллов или среднему баллу (результат будет отличаться, 
если у студентов разное количество оценок)
Реализована проверка заполнения обязательных полей
По нажатию кнопки выгружается 5 лучших и 5 худших студентов. Результаты отсортированы по убыванию суммы оценок или средней оценки (в зависимости
от выбранной опции)

Возможные направления дальнейшей работы над приложением в целом:
- Настройка авторизации и аутентификации
- Улучшение дизайна (UX/UI)
